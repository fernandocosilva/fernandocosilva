## 📌 Projetos de Destaque (Pinados na Home)

Estas são as quatro principais soluções do meu portfólio, selecionadas para demonstrar minha capacidade técnica ponta a ponta:

### 1. 🤖 [Chat-Copilot](https://github.com/fernandocosilva/chat-copilot)
* **Foco:** Engenharia de IA Avançada & Arquitetura Corporativa
* **Descrição:** Assistente inteligente baseado em Large Language Models (LLMs) que utiliza o SDK **Semantic Kernel** da Microsoft. O projeto simula um copiloto de produtividade empresarial dividido em uma SPA React, uma API .NET e um Worker Service assíncrono para ingestão de memória semântica (vetorização).
* **Diferencial:** Aplicação de engenharia de prompt e janelas de contexto estritas para mitigação de alucinações e governança de acesso via Azure Active Directory.

### 🎮 2. [FinQuest — RPG de Educação Financeira](https://github.com/fernandocosilva/dio-lab-vibe-coding-app-financas)
* **Foco:** Engenharia de Prompt, Visão de Produto & Negócio Bancário
* **Descrição:** Aplicativo mobile gamificado voltado para o diagnóstico e triagem financeira de jovens. Desenvolvido no Desafio Vibe Coding da DIO através de orquestração de prompts complexos de alta fidelidade (PRD) injetados em ferramentas de geração autônoma de código (**Lovable**).
* **Diferencial:** Tradução de regras de negócio de crédito reais e alocação de ativos (Gráfico de Pizza) em uma jornada interativa com pontuação e níveis automatizados em memória local.

### 📊 3. [DataAnalyser](https://github.com/fernandocosilva/DataAnalyser)
* **Foco:** Ciência de Dados, Estatística & Resiliência Algorítmica
* **Descrição:** Aplicação desktop em Python + Tkinter para análise rápida de datasets locais ou remotos (integração via cache com Kaggle API). Entrega matrizes de correlação de Pearson, gráficos de regressão linear em alta definição (DPI=300) e histogramas automáticos.
* **Diferencial:** Implementação de arquitetura de **degradação suave (fallback)**. Caso a biblioteca SciPy não esteja no ambiente, o sistema calcula manualmente a Função de Distribuição Acumulada de Gauss por meio da Função de Erro Matemático (`math.erf`), blindando o app contra falhas.

### 🎟️ 4. [Rifa_Machine](https://github.com/fernandocosilva/Rifa_Machine)
* **Foco:** Lógica Pura, Integridade Transacional & Controle de Estado
* **Descrição:** Sistema completo de gerenciamento e auditoria de bilhetes numerados com interface gráfica. Garante que um ativo finito nunca sofra dupla alocação (venda duplicada), permitindo transferências seguras de propriedade e sorteios perfeitamente aleatórios.
* **Diferencial:** Uso de compressão por conjuntos (*set comprehension*) em tempo de execução para higienização e desduplicação de inputs de dados e sincronização reativa de estados multi-janelas (`tk.Toplevel`).
